# **📱 APP 企劃規格書：都會區 YouBike 2.0 供需動態預測專家 (EcoBike Insight)**

本 APP 旨在將複雜的 YouBike 2.0 數據轉化為直觀的「供需預報系統」，幫助通勤族與學生掌握特定站點的車輛動態，避免面臨「無車可借」或「無位可還」的窘境。

## **🛠️ 技術棧與核心套件 (Tech Stack)**

為實現 APP 的數據處理與可視化功能，後端與資料層將整合以下 Python 核心套件：

| 套件名稱 | 在 APP 中扮演的角色 |
| :---- | :---- |
| **requests** | **即時資料同步**：負責向政府開放資料平台（Open Data）發送 API 請求，即時抓取最新站點資訊 。 |
| **pandas** | **數據核心引擎**：負責 APP 內部的表格化資料處理，進行資料清洗、過濾、排序與遺失值處理 。 |
| **matplotlib / seaborn** | **動態圖表渲染**：將歷史供需數據轉化為 APP 介面上的折線圖、熱力圖等視覺化圖表 。 |
| **schedule / time** | **背景排程服務** |

## **⚙️ 後端數據與核心邏輯 (Backend & Logic)**

APP 的後端運作分為三個核心階段，確保資料從採集到呈現的流暢性：

### **階段 1：API 串接與自動化資料資料庫 (Data Collection Service)**

* **即時資料抓取**：後端系統調用 requests.get() 讀取縣市政府（如台北市、台中市）開放資料平台的 YouBike 2.0 JSON 格式 API 。  
* **擷取欄位**：站點名稱、總車柱數、目前可用車輛、目前空柱數 。  
* **定時排程機制**：  
  💡 **核心邏輯**：由於官方 API 僅提供「當下」狀態，後端需執行背景迴圈並搭配 time.sleep()，設定**每 10 分鐘自動抓取一次**資料 。  
* **資料封存**：每次抓取的資料皆會自動附帶「抓取當下的時間戳記（Timestamp）」，並以 Append（附加）形式寫入內部 CSV 檔案或資料庫中，累積至少 3 天至 1 週的基底數據 。

### **階段 2：數據清洗與用戶客製化篩選 (Data Processing Pipe)**

* **高效載入**：使用 pandas.read\_csv() 讀取封存的歷史供需資料 。  
* **精準站點過濾**：考慮到全縣市站點過於龐大，APP 將提供「**我的最愛/常用站點**」功能，讓用戶挑選 **3 到 5 個具代表性的特定站點**（例如：特定高中門口、最近的捷運站、附近的公園）進行追蹤分析 。  
* **容錯與補值機制**：當遇到網路斷線導致資料缺失時，系統會自動在 DataFrame 中進行補值或剔除異常值，確保圖表不中斷 。

## **🎨 APP 介面功能與視覺化設計 (Frontend & UI/UX)**

本 APP 的前端核心在於「讓數字說話」，透過直觀的圖表協助用戶找出供需規律 。

### **功能 1：動態時間序列折線圖 (Time-Series Dashboard)**

* **介面呈現**：整合 matplotlib 渲染之圖表，**X 軸為時間軸，Y 軸為剩餘車輛數** 。  
* **智慧對比模式**：  
  * 支援同時勾選兩個站點進行「供需對比」 。  
  * *範例情境*：同時觀察「捷運站」與「高中門口」，用戶可直觀看到兩者的波峰與波谷剛好相反（例如：早上 7:30 捷運站的車被騎光，而學校門口的車柱被停滿），藉此調整出門時間 。

### **功能 2：特殊事件與天氣預報模組 (Weather & Event Insights)**

* **天候租借率對比**：結合天氣 API，系統會對比「晴天」與「雨天」的歷史租借率差異 。  
* **出門建議**：若系統偵測到當前為雨天，會自動調出歷史雨天數據，提示用戶該站點可能剩餘較多車輛（或因下雨天搭公車人數變多而車輛完好），提供更聰明的通勤決策 。
